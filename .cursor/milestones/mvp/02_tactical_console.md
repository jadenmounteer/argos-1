# Tactical Console (The Interface)

## Goal

Build the React LCARS-inspired UI and integrate the Web Speech API for "Wake Word" and command transcription.

### Tasks

Note on Browser Compatibility: The Web Speech API is most stable in Chrome/Edge. For the MVP, ensure the UI has a "Manual Override" (a text input box) in case the microphone environment is noisy or the browser limits the API.

Note on State: Keep the speech-to-text state separate from the AI-response state to prevent the "feedback loop" where ARGOS reviews its own spoken words.

#### [ ] Task 2: Web Speech Integration (The Ears)

Implement a useArgosVoice hook utilizing the window.SpeechRecognition (or webkitSpeechRecognition) API.

Create a visual "Listening" indicator that pulses when the microphone is active.

#### [ ] Task 3: Real-Time Stream Consumer (The Data Link)

Implement a service to consume the Spring Boot /api/v1/command endpoint.

Pro Tip: Use Server-Sent Events (SSE) or a simple Fetch stream reader to display text as it is generated by the local LLM.

Acceptance: The UI must not "freeze" while waiting for the full response.

Implement an "Auto-scroll" feature so the logs feel like a live readout.
